# Skill — Mudança segura

## Quando usar

Use esta skill para qualquer alteração funcional com risco moderado ou alto.

## Procedimento

1. Preencher `task-brief`.
2. Classificar fatos, hipóteses, riscos e pendências.
3. Mapear módulos e contratos impactados.
4. Escrever testes primeiro.
5. Implementar mudança mínima.
6. Executar checklist de revisão.
7. Gerar handoff compacto.

## Perguntas de controle

- A mudança toca tenant?
- A mudança toca autorização?
- A mudança precisa de auditoria?
- Existe risco de acoplamento novo?
- README/CHANGELOG/ADR precisam atualização?
